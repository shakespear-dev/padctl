```
  T1  community device configs + padctl --validate
```

**Changes**:

`src/tools/validate.zig` — seven-pass static validator:

```zig
pub const ValidationError = struct {
    file: []const u8,
    message: []const u8,
};

pub fn validate(path: []const u8, allocator: std.mem.Allocator) ![]ValidationError
```

Passes 1–2 are handled by the existing `config.device` parser returning errors. Passes 3–7 iterate over the parsed `DeviceConfig` structure. All errors collected into a slice; validate does not stop at first error.

CLI:
```
padctl --validate <file> [<file> ...]
```

Exit 0 = all valid. Exit 1 = at least one validation error. Exit 2 = file not found / parse failure.

`CONTRIBUTING.md` documents three-step contribution flow:
1. Run `padctl-capture` to produce TOML skeleton.
2. Complete field names and semantics manually.
3. Open PR; CI runs `padctl --validate`; human review; merge.

**Tests (L0)**:
- Valid TOML with all required fields → 0 errors, exit 0
- Missing `[device] pid` → schema error reported
- Field `offset = 60, type = "u32le"` in 64-byte report → pass (60+4=64, boundary OK)
- Field `offset = 61, type = "u32le"` in 64-byte report → offset-bounds error
- Two fields with same name in same `[[report]]` → uniqueness error
- Button with `bit_index = 8` in single-byte group → bit-range error
- Two `[[report]]` with identical `match` bytes → overlap error
- `checksum.algo = "md5"` → unknown-algo error

**Tests (L1)**:
- DualSense 64-byte USB report (with byte 0 = 0x01, sticks, triggers, buttons, IMU) → `interpreter.processReport()` returns non-null delta with correct `left_x`, `gyro_x`, button bits
- Switch Pro BT report (49 bytes, report id 0x30) → all button and axis fields parse correctly

```yaml
strategy:
  matrix:
    target: [x86_64-linux-musl, aarch64-linux-musl]

  - zig build -Doptimize=ReleaseSafe -Dtarget=${{ matrix.target }}
  - strip zig-out/bin/padctl
  - tar czf padctl-${{ github.ref_name }}-${{ matrix.target }}.tar.gz \

        --add-file devices/ --add-file contrib/ --add-file sdk/plugin.h
```

`contrib/aur/PKGBUILD`:

`contrib/aur/padctl-bin/PKGBUILD`:
- Fetches prebuilt tarball from GitHub Release for the current arch.
- No build step; no `makedepends`.

`flake.nix` — zig2nix-based, `x86_64-linux` + `aarch64-linux`, `packages.default` + `devShells.default`.

## T4: WASM Plugin Runtime

**Files**:
- `src/wasm/runtime.zig` (new)
- `src/wasm/host_functions.zig` (new)
- `src/wasm/plugin.zig` (new)

- `devices/sony/dualsense.wasm` (new, reference implementation)
- `sdk/plugin.h` (new)

**Changes**:

`src/wasm/runtime.zig`:

```zig
pub const Runtime = struct {
    m3_env: *M3Environment,
    m3_runtime: *M3Runtime,

    pub fn init(allocator: std.mem.Allocator, memory_limit: usize) !Runtime
    pub fn deinit(self: *Runtime) void
    pub fn loadModule(self: *Runtime, wasm_bytes: []const u8) !Module
};
```

Memory limit set via `m3_runtime->memoryLimit`. wasm3 is linked as a static C library via `exe.addCSourceFiles()`.

`src/wasm/host_functions.zig` — registers six host functions into the wasm3 runtime environment. Each validates ptr+len bounds before accessing linear memory.

`src/wasm/plugin.zig`:

```zig
pub const Plugin = struct {
    runtime: Runtime,
    fn_init_device: ?*M3Function,
    fn_process_calibration: ?*M3Function,
    fn_process_report: ?*M3Function,
    device_io: *DeviceIO,              // borrowed, not owned

    pub fn load(path: []const u8, device_io: *DeviceIO, allocator: std.mem.Allocator) !Plugin
    pub fn deinit(self: *Plugin) void
    pub fn initDevice(self: *Plugin) !void          // calls fn_init_device if present; 5 s watchdog
    pub fn processCalibration(self: *Plugin, buf: []const u8) !void
    pub fn processReport(self: *Plugin, raw: []const u8, out: []u8) !usize  // returns bytes written
};
```

Optional exports: missing `init_device` / `process_calibration` → silently skipped. Missing `process_report` when declared in TOML → validation warning at startup.

Timeout watchdog: `timerfd_create(CLOCK_MONOTONIC)` set before each hook call; on expiry, `pthread_cancel` on the WASM execution thread; padctl resumes normally.

`sdk/plugin.h` — C header (import module `"env"`):
```c
// Host functions (provided by padctl)
extern int32_t device_read(int32_t report_id, int32_t buf_ptr, int32_t buf_len);
extern int32_t device_write(int32_t buf_ptr, int32_t buf_len);
extern void    log(int32_t level, int32_t msg_ptr, int32_t msg_len);
extern int32_t get_config(int32_t key_ptr, int32_t key_len, int32_t val_ptr, int32_t val_cap);
extern void    set_state(int32_t key_ptr, int32_t key_len, int32_t val_ptr, int32_t val_len);
extern void    abort(int32_t msg_ptr, int32_t msg_len);

// Plugin exports (implemented by plugin)
int32_t init_device(void);
int32_t process_calibration(int32_t cal_buf_ptr, int32_t cal_buf_len);
int32_t process_report(int32_t raw_ptr, int32_t raw_len,
                       int32_t out_ptr, int32_t out_len);
```

**Tests (L1)**:
- Minimal WASM module exporting `init_device` returning 0 → `plugin.initDevice()` returns ok
- WASM module with infinite loop in `init_device` → watchdog fires within 5 s + 100 ms margin; `initDevice()` returns error
- WASM module writing to address 2 MB (beyond 1 MB limit) → wasm3 trap; `processReport()` returns error; no crash
- WASM module missing `init_device` export → `load()` succeeds; `initDevice()` is a no-op
- `process_calibration` hook → host `get_config("gyro_x.offset", ...)` returns value matching device config field

## T5: Output DSL Emulate Extension

**Files**:
- `src/output.zig` (extend)
- `src/config/device.zig` (extend to parse `[output]` and `[output.capabilities]`)
- `engineering/output.md` (update spec)

**Changes**:

`config/device.zig` additions:

```zig
pub const OutputCapabilities = struct {
    axes:    ?[]AxisCode,    // EV_ABS codes
    buttons: ?[]ButtonCode,  // EV_KEY codes
    rumble:  bool = false,
};

pub const OutputConfig = struct {
    emulate:      ?[]const u8 = null,   // preset name; resolved before vid/pid/name are used
    vid:          ?u16 = null,
    pid:          ?u16 = null,
    name:         ?[]const u8 = null,
    capabilities: ?OutputCapabilities = null,
};
```

`DeviceConfig.output: ?OutputConfig` — null if `[output]` section absent.

`src/output.zig` — `initUinput()` reads `config.output`:
- If `output.vid` set → use in `uinput_setup.id.vendor`
- If `output.capabilities.axes` set → register only declared axes; else infer from `[input]`
- If absent → existing behaviour unchanged

**Tests (L1)**:
- TOML with `[output] vid = 0x054c, pid = 0x0ce6` → `uinput_setup.id.vendor = 0x054c, id.product = 0x0ce6`
- TOML with `[output.capabilities] axes = ["ABS_X"]` → mock uinput receives `UI_SET_ABSBIT` only for `ABS_X`
- TOML without `[output]` → uinput setup identical to Phase 3 baseline (regression guard)
- `[output.capabilities] buttons = ["BTN_SOUTH", "BTN_EAST"]` → `UI_SET_KEYBIT` called for both and only those codes
